Section 1: Core Concepts Explained

Before we dive into the questions, let's establish a solid foundation by explaining the core concepts of arrays, linked lists, and strings.

Arrays

An array is a data structure that stores a collection of elements, each identified by an index or key. Arrays can be one-dimensional or multi-dimensional, and their elements can be of the same data type or different data types. In programming, arrays are used to store and manipulate collections of data, making them a fundamental data structure in computer science.

Linked Lists

A linked list is a data structure that consists of nodes, each containing a value and a reference (i.e., a "link") to the next node in the list. Linked lists can be singly linked, where each node only points to the next node, or doubly linked, where each node points to both the previous and next node. Linked lists are useful for inserting, deleting, and traversing elements efficiently.

Strings

A string is a sequence of characters, such as letters, digits, or symbols. In programming, strings are used to represent text, and they can be manipulated using various operations, such as concatenation, substring extraction, and searching.

Which of the following operations is performed more efficiently by a doubly linked list than by a singly linked list?

The correct answer is: deleting a node. In a doubly linked list, deleting a node is more efficient because we can traverse both forward and backward to find the node and update the links. In a singly linked list, we would need to traverse from the beginning to find the node, making deletion less efficient.

In a max-heap, the value of each node is:

Write pseudocode to delete the middle node of a singly linked list.

The correct answer is: 

function deleteMiddleNode(node head) {
  if head is null then
    return
  end if

  node slow = head
  node fast = head

  while fast is not null and fast.next is not null {
    slow = slow.next
    fast = fast.next.next
  }

  node temp = slow.next
  slow.next = temp.next
  free(temp)
}
This pseudocode uses the slow and fast pointer technique to find the middle node and then deletes it by updating the links.
